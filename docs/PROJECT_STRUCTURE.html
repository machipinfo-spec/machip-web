主要フォルダの詳細説明

.vscode DIR

VS Codeでの開発用の共有設定フォルダ。エディタ共通設定の settings.json などが置かれています。

docs DIR

仕様書やガイドなどの関連ドキュメントを配置するフォルダ。E2Eテスト仕様に関する e2e-test フォルダなどが含まれます。

e2e DIR

Playwright によるエンドツーエンド（E2E）テストコード群。以下の構造でテストが整理されています：

auth.setup.ts: 認証フェーズの事前共通処理
guest-mode/: 未ログイン（ゲスト）状態のテストケース
normal-mode/: 通常ログイン状態のテストケース
playwright DIR

Playwrightの実行に関わる内部情報フォルダ。.auth/user.json 等にログインセッション情報が一時的にキャッシュされます。

public DIR

直接Web公開される画像やファビコン、各種静的ファイル。Firebaseプッシュ通知用の firebase-messaging-sw.js（サービスワーカー）も含みます。

src DIR

アプリケーションのメインロジックを構成する、最重要フォルダです：

app/: Next.js App Router 規格のページ構造とルーティング設計
components/: プロジェクト全体で利用可能な共通UIパーツ群
contexts/: React Context API による全域の状態管理ロジック
features/: 各機能ドメインに特化したロジックとビューの集まり
hooks/: リアクティブ設計のためのカスタムフック群
lib/: 各種プラグインや初期設定の仲介モジュール
services/: Cognito（認証）やFirebase（データベース/FCM）等との連携 API
types/: TypeScript の共有型定義ファイル
utils/: 計算や日付処理などの共通ユーティリティ（純粋関数）
middleware.ts: パスチェックや認証制御のリダイレクトミドルウェア

設定ファイル・ルートドキュメント

.gitignore FILE

Gitリポジトリ管理から除外するディレクトリおよびファイルを定義します。

amplify.yml FILE

AWS Amplify ホスティングにおける CI/CD ビルド手順、キャッシュポリシー、デプロイディレクトリを指定します。

eslint.config.mjs FILE

コード品質およびフォーマットを統一するための ESLint 設定ファイル（v9 Flat Config形式）。

next.config.ts FILE

Next.js のビルドオプション、画像ドメイン許可設定などをコントロールします。

package.json FILE

プロジェクトの名称、実行スクリプト（dev, build 等）、および依存ライブラリのバージョンリスト。

playwright.config.ts FILE

E2Eテストフレームワーク Playwright のブラウザ設定、並列処理の有無、ローカル開発サーバー起動手順などの定義。

tsconfig.json FILE

TypeScriptのコンパイル設定。パスエイリアス（@/*）や strict モードなどのルール設定。

Git除外対象（.gitignore）

機密情報の流出防止や、ビルド生成ファイルによる容量肥大化を避けるため、以下のリソースは Git 管理から明示的に除外されています。

node_modules/
npm パッケージ依存関係の実体。
.next/ , out/ , build/
ビルドにより生成される成果物や一時キャッシュ。
.env.local / .env*
各種APIキーや認証資格情報などの秘密キー環境変数。
*.tsbuildinfo
TypeScript のインクリメンタルコンパイル用のキャッシュデータ。
next-env.d.ts
Next.js がビルド時に自動生成する型情報ファイル。
.git/
Git の変更履歴・管理メタデータ。
