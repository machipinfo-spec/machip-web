💡

Next.js 環境変数の露出仕様について

Next.js では、環境変数のプレフィックス（接頭辞）によってセキュリティスコープが厳格に区別されます。

  • NEXT_PUBLIC_ プレフィックスあり (Client & Server):
    ビルド時にブラウザ用 JavaScript コードへ直接埋め込まれます。クライアントサイド（コンポーネントやカスタムフック）で安全にアクセスできます。APIキーなどの公開可能な情報にのみ使用してください。
  • プレフィックスなし (Server Only):
    サーバーサイド（Next.js API Routes, Server Actions, Middleware, `getStaticProps`等）のみでアクセス可能です。クライアント側のブラウザには一切送信されません。CognitoのClient SecretやAPI暗号化キーなどの機密情報を安全に隠蔽するために使用します。

1. NextAuth & AWS Cognito 認証関連

NextAuth.jsとAWS Cognitoを連携して安全なJWTセッション認証および更新フローを行うための設定です。

キー名 スコープ 説明 設定例 (ローカル)
COGNITO_ISSUER Server CognitoユーザープールのエンドポイントURL（認証トークン発行元） https://cognito-idp.ap-northeast-1.amazonaws.com/ap-northeast-1_XXXXX
COGNITO_CLIENT_ID Server Cognito ユーザープール内に作成されたアプリクライアントのID 7hxxxxxxxxxxxxxxxxxxxx
COGNITO_CLIENT_SECRET Server Cognitoアプリクライアントのシークレット（※トークンリフレッシュ時等に必須） 1vxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NEXTAUTH_URL Server NextAuthがリダイレクトやコールバック基準として使用するアプリのベースURL http://localhost:3000
AUTH_SECRET Server NextAuthがセッションCookieを暗号化するために使用するランダムな秘密文字列 b6d... (openssl rand -base64 32 等で生成)
AUTH_TRUST_HOST Server VercelやAmplifyなどプロキシ環境下でホストの信頼を強制するフラグ true
NEXTAUTH_TRUST_HOST Server （下位互換/特定ライブラリ用）ホスト信頼フラグ true

環境変数の設定手順と注意点

ローカル開発環境での準備

ローカル開発環境では、プロジェクトのルートディレクトリ直下に .env.local ファイルを作成し、上記の環境変数を記述します。 

# AWS Cognito
COGNITO_ISSUER=https://cognito-idp.ap-northeast-1.amazonaws.com/ap-northeast-1_XXXXX
COGNITO_CLIENT_ID=xxxxxxxxxxxxxxxxxxxxxx
COGNITO_CLIENT_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# NextAuth
NEXTAUTH_URL=http://localhost:3000
AUTH_SECRET=openssl_rand_base64_string_here
AUTH_TRUST_HOST=true

# Firebase & FCM
NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=project-id.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=project-id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=project-id.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789012
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789012:web:a1b2c3d4e5f6
NEXT_PUBLIC_FIREBASE_VAPID_KEY=B...

# Google Maps
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=AIzaSy...
NEXT_PUBLIC_GOOGLE_MAP_ID=a1b2c3d4e5f67890

# API Endpoint
NEXT_PUBLIC_API_URL=http://localhost:3000
【最重要】セキュリティとGit管理

.env.local および .env に関連するファイルは、秘密鍵や認証シークレットが記載されているため、絶対にGit管理（リモートリポジトリへのプッシュ）を行わないでください。 既に .gitignore で保護されていることを確認済みですが、手動でのコミット強制等は絶対に行わないよう厳守してください。

AWS Amplify (ホスティング) での設定方法

本番・検証用サーバーである AWS Amplify Console にデプロイする際は、Amplifyの管理画面の「アプリの設定」 > 「環境変数」にて登録を行います。 ビルド時（`next build`）に NEXT_PUBLIC_ プレフィックスの付く環境変数を埋め込むために、コンソールでの変数の正確な指定が必要です。
